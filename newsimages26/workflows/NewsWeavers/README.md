# MediaEval NewsImages 2026 Code

This repository contains a reproducible workflow for generating MediaEval NewsImages-style editorial images from article titles. The project compares title-only visual planning with and without LoRA fine-tuning, and generates final submission images using SDXL and FLUX.1-schnell.

The submitted runs compare two prompt-writing strategies and two image generators:

1. **Qwen3BPlanPromptZeroShotSDXL**: zero-shot Qwen2.5-3B-Instruct visual planner + SDXL
2. **Qwen3BPlanLoRASDXL**: LoRA-tuned Qwen2.5-3B-Instruct visual planner + SDXL
3. **Qwen3BPlanPromptZeroShotFLUX**: zero-shot Qwen2.5-3B-Instruct visual planner + FLUX.1-schnell
4. **Qwen3BPlanLoRAFLUX**: LoRA-tuned Qwen2.5-3B-Instruct visual planner + FLUX.1-schnell

For each article title, the planner produces a structured JSON with title understanding, a visual generation plan, SDXL prompt, negative prompt, and ranking caption. The LoRA variant is trained with silver labels generated by a Qwen2.5-7B-Instruct teacher that also sees VLM-parsed facts from one paired training image per article. The trained student and zero-shot inference use the article title only.

Since Qwen2.5-3B is a small model and sometimes produces malformed structured output, the inference script includes fuzzy key mapping and JSONL repair to recover near-valid outputs. FLUX.1-schnell does not support a negative prompt; the `negative_prompt` field from the planner output is ignored. Instead, a fixed style phrase is optionally prepended or appended to the positive prompt.

## Required Models

| Role | Model |
| ------ | ------- |
| Visual planner (student, inference) | `Qwen/Qwen2.5-3B-Instruct` |
| Silver label teacher (SFT data only) | `Qwen/Qwen2.5-7B-Instruct` |
| VLM image fact extractor (SFT data only) | `Qwen/Qwen2.5-VL-3B-Instruct` |
| Image generator | `stabilityai/stable-diffusion-xl-base-1.0` |
| Image generator | `black-forest-labs/FLUX.1-schnell` |

## Workflow overview

### Step 1: Extract visible facts from paired training images

Uses `Qwen2.5-VL-3B-Instruct` to extract conservative image-only visible facts from the training images. The output JSONL (`train_image_visible_facts.jsonl`) is used as weak supervision in Step 2.

```bash
python -u scripts/extract_training_image_visible_facts.py \
  --image_dir data/news_images \
  --output_dir outputs/qwen_image_visible_facts \
  --resume
```

### Step 2: Build teacher silver labels

Uses `Qwen2.5-7B-Instruct` to generate structured JSON silver labels from article titles and VLM-parsed image facts.

```bash
TOKENIZERS_PARALLELISM=false python -u scripts/prepare_sft_data.py make-silver \
  --articles data/news_articles.csv \
  --vlm outputs/qwen_image_visible_facts/train_image_visible_facts.jsonl \
  --out outputs/sft/teacher_silver.jsonl \
  --teacher-model Qwen/Qwen2.5-7B-Instruct \
  --title-col article_title \
  --article-id-col article_id \
  --image-col image_id \
  --max-new-tokens 1800 \
  --retries 2 \
  --repetition-penalty 1.0 \
  --no-repeat-ngram-size 0 \
  --temperature 0 \
  --load-in-4bit \
  --local-files-only \
  --save-raw \
  --save-vlm-parse \
  --resume
```

### Step 3: Validate and filter silver labels

```bash
python scripts/prepare_sft_data.py validate-filter \
  --input outputs/sft/teacher_silver.jsonl \
  --valid-out outputs/sft/teacher_silver.valid.jsonl \
  --bad-out outputs/sft/teacher_silver.bad.jsonl
```

Optional manual audit of a random sample:

```bash
python scripts/prepare_sft_data.py sample-audit \
  --input outputs/sft/teacher_silver.valid.jsonl \
  --out outputs/sft/manual_audit_100.jsonl \
  --n 100 \
  --seed 42
```

### Step 4: Build SFT JSONL

```bash
python scripts/prepare_sft_data.py make-sft \
  --input outputs/sft/teacher_silver.valid.jsonl \
  --out outputs/sft/qwen3b_plan_sft.jsonl \
  --require-vlm
```

### Step 5: Split train/validation data

```bash
python scripts/split_sft_train_val.py \
  --input outputs/sft/qwen3b_plan_sft.jsonl \
  --train-out outputs/sft/qwen3b_plan_sft.train.jsonl \
  --val-out outputs/sft/qwen3b_plan_sft.val500.jsonl \
  --val-size 500 \
  --seed 42
```

### Step 6: Train Qwen2.5-3B LoRA

```bash
TOKENIZERS_PARALLELISM=false python -u scripts/train_qwen3b_plan_lora_sft.py \
  --train-file outputs/sft/qwen3b_plan_sft.train.jsonl \
  --eval-file outputs/sft/qwen3b_plan_sft.val500.jsonl \
  --out-dir outputs/qwen3b_plan_lora \
  --base-model Qwen/Qwen2.5-3B-Instruct \
  --local-files-only \
  --load-in-4bit \
  --max-seq-length 2048 \
  --num-train-epochs 2 \
  --learning-rate 1e-4 \
  --per-device-train-batch-size 1 \
  --per-device-eval-batch-size 1 \
  --gradient-accumulation-steps 8 \
  --lora-r 32 \
  --lora-alpha 64 \
  --lora-dropout 0.05 \
  --logging-steps 10 \
  --save-steps 200 \
  --eval-steps 200 \
  --save-total-limit 3
```

### Step 7: Run inference on test titles

First, convert the test CSV to JSONL.

```bash
python scripts/convert_csv_to_jsonl.py \
  --input data/news_articles_test.csv \
  --output data/test_titles.jsonl
```

To export only a random subset (e.g. for a quick sanity check), add `--sample 50 --seed 42`. To export only the first N rows, use `--first N`.

**Zero-shot** (no adapter):

```bash
TOKENIZERS_PARALLELISM=false python -u scripts/infer_qwen3b_visual_planner.py \
  --base-model Qwen/Qwen2.5-3B-Instruct \
  --articles data/test_titles.jsonl \
  --out outputs/infer/qwen3b_zeroshot_test.jsonl \
  --title-col article_title \
  --article-id-col article_id \
  --image-col image_id \
  --max-new-tokens 1200 \
  --retries 2 \
  --repetition-penalty 1.08 \
  --no-repeat-ngram-size 5 \
  --load-in-4bit \
  --local-files-only
```

**LoRA** (with adapter):

```bash
TOKENIZERS_PARALLELISM=false python -u scripts/infer_qwen3b_visual_planner.py \
  --base-model Qwen/Qwen2.5-3B-Instruct \
  --adapter outputs/qwen3b_plan_lora \
  --articles data/test_titles.jsonl \
  --out outputs/infer/qwen3b_lora_test.jsonl \
  --title-col article_title \
  --article-id-col article_id \
  --image-col image_id \
  --max-new-tokens 1200 \
  --retries 2 \
  --repetition-penalty 1.08 \
  --no-repeat-ngram-size 5 \
  --temperature 0 \
  --load-in-4bit \
  --local-files-only \
  --resume
```

### Step 8: Generate SDXL images

Replace `--approach-name` with the run name for each submission variant (e.g. `Qwen3BPlanPromptZeroShotSDXL` for zero-shot, `Qwen3BPlanLoRASDXL` for LoRA).

```bash
python -u scripts/sdxl_generate_images.py \
  --input outputs/infer/qwen3b_lora_test.jsonl \
  --output-dir outputs/sdxl_qwen3b_lora \
  --input-mode exported \
  --variants chosen \
  --approach-name Qwen3BPlanLoRASDXL \
  --group-name NewsWeavers \
  --submission-root outputs/mediaeval_submission \
  --model stabilityai/stable-diffusion-xl-base-1.0 \
  --local-files-only \
  --images-per-prompt 1 \
  --steps 30 \
  --guidance-scale 6.5 \
  --width 920 \
  --height 520 \
  --scheduler dpm \
  --seed 42
```

### Step 9: Generate FLUX images

Replace `--approach-name` with the run name for each submission variant (e.g. `Qwen3BPlanPromptZeroShotFLUX` for zero-shot, `Qwen3BPlanLoRAFLUX` for LoRA).

```bash
PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True \
python -u scripts/flux_generate_images.py \
  --input outputs/infer/qwen3b_lora_test.jsonl \
  --output-dir outputs/flux_qwen3b_lora \
  --input-mode exported \
  --variants chosen \
  --approach-name Qwen3BPlanLoRAFLUX \
  --group-name NewsWeavers \
  --submission-root outputs/mediaeval_submission \
  --model black-forest-labs/FLUX.1-schnell \
  --local-files-only \
  --sequential-cpu-offload \
  --images-per-prompt 1 \
  --steps 4 \
  --guidance-scale 0.0 \
  --max-sequence-length 256 \
  --style-profile symbolic \
  --style-position prefix \
  --width 736 \
  --height 416 \
  --seed 42
```

## Reproducibility notes

- The teacher (Step 2) sees title + VLM-parsed image facts. The student (Steps 7–9) sees the title only.
- All runs use `--seed 42` where applicable.
- SDXL output resolution is 920×520 (downsampled to 460×260 for MediaEval submission).
- FLUX output resolution is 736×416 (downsampled to 460×260 for MediaEval submission).
